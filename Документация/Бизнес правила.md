## Бизнес-правила
(См. также правила для данного приложения в дополнительной спецификации.)  

| ИД | Правило | Частота изменения | Источник |
|----|---------|-----------------|----------|
| ПРАВ1 | Каждый документ должен иметь уникальное имя в пределах текущей сессии редактора | Низкая вероятность изменения | Архитектурные требования редактора |
| ПРАВ2 | Любые изменения текста должны быть возможны только при открытом документе | Средняя вероятность изменения | Логика работы Editor |
| ПРАВ3 | Подсветка синтаксиса должна применяться только к поддерживаемым языкам | Средняя вероятность изменения, при добавлении новых языков | Технические требования проекта |
| ПРАВ4 | Курсор может находиться только на существующих элементах документа (Letter, Word, Sentence, Paragraph) | Низкая вероятность изменения | Логика работы CompositeManager и Document |
| ПРАВ5 | При экспорте документа в PDF или JSON необходимо сохранять структуру и содержимое без потерь | Средняя вероятность изменения | Требования к ExportManager |
| ПРАВ6 | Печать документа должна использовать настройки PrintSettings и соответствовать формату документа | Средняя вероятность изменения | Технические требования PrintManager |
| ПРАВ7 | Пользователь не может удалять последний абзац в документе, чтобы сохранить минимальную структуру | Низкая вероятность изменения | Логика безопасности документа |
| ПРАВ8 | При поиске текста или выделении подсветки должны корректно обрабатываться вложенные элементы (Word → Letter) | Средняя вероятность изменения | Логика работы CompositeManager |
| ПРАВ9 | Ошибки синтаксиса должны отображаться пользователю в Editor с указанием позиции и сообщения | Низкая вероятность изменения | Технические требования SyntaxHighlighter |
| ПРАВ10 | Документ может быть закрыт только после сохранения всех несохранённых изменений или подтверждения пользователя | Низкая вероятность изменения | Требования к надежности системы |
